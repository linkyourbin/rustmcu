# 主页空白问题修复

## ❌ 问题原因

主页点击后变空白的原因：

**homeInfoParams 模式的局限**：
- ✅ 可以显示标题和介绍
- ❌ 但不会显示文章列表
- ❌ 点击后只有顶部信息，下方空白
- ❌ 用户体验不好

---

## ✅ 最终解决方案

**恢复使用 profileMode**（主题最稳定的模式）：

### 配置改回 profileMode

```yaml
profileMode:
  enabled: true
  title: "🦀 RUST MCU"
  subtitle: |
    探索 Rust 嵌入式开发 | STM32 · ESP32 · RP2040 · PY32
  imageUrl: "profile/avatar.svg"
  imageWidth: 120        # 缩小头像
  imageHeight: 120
  buttons:
    - name: 🚀 MCU平台
      url: /mcu-platforms/
    - name: 📖 开始学习
      url: /categories/home/
    - name: 🔍 归档
      url: /archives/
    - name: 💡 关于
      url: /about/
```

### 优化点

1. **头像缩小** - 从 360x360 改为 120x120
2. **简洁副标题** - 只保留关键信息
3. **4个按钮** - MCU平台、学习、归档、关于
4. **无动画** - 静态简洁
5. **显示文章** - profileMode 模式下正常显示

## 💡 为什么不显示文章列表？

**这是 profileMode 的设计**：

### profileMode 特点
- ✅ 主页只显示个人信息卡片
- ✅ 简洁专业的门户页
- ✅ 引导用户点击按钮进入内容
- ❌ 不在主页显示文章列表

### 如果想要主页显示文章

有3个选择：

#### 选项1: 使用 list 模式（推荐）
```yaml
# 不配置 profileMode 或 homeInfoParams
# 主页会显示文章列表
```

#### 选项2: 使用 regular 模式
```yaml
params:
  homeInfoParams:
    Title: "标题"
    Content: "内容"
```
但这个模式有bug（就是你遇到的空白问题）

#### 选项3: 混合模式
在其他页面放文章，主页保持简洁

## 📊 模式对比

### profileMode（当前使用）✅
```
主页: 个人卡片（简洁门户）
文章: 通过按钮/导航访问
优点: 专业、简洁、稳定
缺点: 主页不直接显示文章
```

### homeInfoParams（有问题）❌
```
主页: 信息卡片 + 空白

优点: 无
缺点: 显示空白
```

### list 模式（可选）
```
主页: 文章列表
文章: 直接显示
优点: 信息密度高
缺点: 缺少个人信息展示
```
